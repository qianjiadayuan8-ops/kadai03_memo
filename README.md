# ①課題名
消耗品管理アプリ

## ②課題内容（どんな作品か）
- 登録した日用品の在庫を管理し、使い切ったタイミングで「お買い物リスト」に追加できるアプリです。
消耗品と買い物リストを分けて管理することで、日常の買い忘れを防げるようにしました。

## ⑤工夫した点・こだわった点
- ローカルストレージのデータ設計を工夫しました。
「日用品データ」と「買い物リストデータ」を別々のキーで管理し、それぞれ配列をJSON形式に変換して保存しています。
データを役割ごとに分離することで、管理しやすく拡張しやすい構成を意識しました。

## ⑥難しかった点・次回トライしたいこと（又は機能）
- ローカルストレージへ保存したデータが、画面上で期待通りに反映されない問題に何度も苦戦しました。
原因を調べる中で、「データ更新後に再描画が必要である」という点を理解しましたが、状態管理と画面描画の関係を実感として掴むまで時間がかかりました。
